## Summary
- **Objective**: To develop the Agentic RAG framework, which integrates retrieval and generation for complex tasks, balancing performance and computational efficiency through structured architecture, prompt design, and query classification.  
- **Methodologies**: The framework employs three nodes (Orchestrator, Answer Node, RAG Node) with system prompts for task flow and answer generation. A routing system classifies queries using cosine similarity thresholds. Evaluation metrics include classification accuracy (FIQA, CQADupStack-EN) and computational costs (latency, token usage). Framework comparisons (e.g., smolagents, langgraph, autogen) assess implementation trade-offs.  
- **Results**: Qwen3-32B achieved highest accuracy (95.5% on FIQA, 93.8% on CQADupStack-EN) but required 4× A40 GPUs and higher latency. Routing thresholds are critical for query classification accuracy. Thinking mode improved output quality but increased token usage (e.g., 32B model’s output tokens rose from 1109 to 1695). Smaller models (e.g., Qwen3-0.6B) prioritized speed over output length.  
- **Key Contributions**:  
  1. Agentic RAG architecture with three nodes and routing system for query classification.  
  2. Evaluation of model size trade-offs (accuracy vs. computational cost).  
  3. Framework comparison for implementation (e.g., smolagents vs. langgraph).  
  4. Insights into prompt design, threshold tuning, and hardware requirements.  
- **Conclusions**: Agentic RAG balances performance and efficiency, but larger models demand significant resources. Routing thresholds and thinking mode are pivotal for accuracy and output quality. Framework selection depends on simplicity vs. integration needs. Model size, hardware, and prompt design are critical for optimizing Agentic RAG systems.

## 摘要
- **目标**：开发代理增强型RAG框架，通过结构化架构、提示设计和查询分类，在复杂任务中集成检索与生成，平衡性能与计算效率。  
- **方法**：框架采用三个节点（协调器、答案节点、RAG节点）及系统提示实现任务流程与答案生成。路由系统通过余弦相似度阈值对查询进行分类。评估指标包括分类准确率（FIQA、CQADupStack-EN）和计算成本（延迟、令牌使用）。框架对比（如smolagents、langgraph、autogen）分析实现的权衡。  
- **结果**：Qwen3-32B在FIQA上达到最高准确率（95.5%）、CQADupStack-EN上为93.8%，但需4×A40显卡且延迟更高。路由阈值对查询分类准确率至关重要。思考模式提升了输出质量，但增加了令牌使用（如32B模型的输出令牌数从1109增至1695）。较小模型（如Qwen3-0.6B）优先考虑速度而非输出长度。  
- **关键贡献**：  
  1. 三节点与路由系统相结合的代理增强型RAG架构。  
  2. 模型规模权衡的评估（准确率与计算成本）。  
  3. 框架实现的对比分析（如smolagents与langgraph）。  
  4. 提示设计、阈值调优及硬件需求的见解。  
- **结论**：代理增强型RAG在性能与效率间取得平衡，但大模型需大量资源。路由阈值与思考模式对准确率和输出质量至关重要。框架选择取决于简洁性与集成需求。模型规模、硬件及提示设计是优化代理增强型RAG系统的关键。
