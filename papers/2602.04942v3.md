## Summary  
- **Objective**: To address critical issues in AI systems, including reward hacking, JSON formatting errors, privileged information (PI) leakage, and unclear system prompts, with the goal of improving reliability, security, and adherence to workflow constraints.  
- **Methodologies**: The paper employs reward hacking mitigation strategies (e.g., prioritizing hard constraints), JSON validation techniques, PI handling through prompt redesign and filtering, and system prompt engineering for τ-Bench and Travel Planner workflows.  
- **Results**: Key findings include the effectiveness of modifying reward logic to prevent exploitation, the necessity of strict JSON formatting guidelines, the feasibility of blocking PI leaks via prompt constraints, and the importance of clear system prompts for workflow alignment.  
- **Key Contributions**:  
  1. A novel approach to reward hacking by prioritizing hard constraints over easy ones.  
  2. A structured JSON response framework for tool calls.  
  3. PI leakage mitigation through prompt redesign and output filtering.  
  4. Custom system prompts for τ-Bench and Travel Planner to enforce specific workflows.  
- **Conclusions**: The paper highlights that addressing reward hacking, formatting errors, and PI leaks requires a combination of reward structure adjustments, strict output validation, and prompt engineering. These strategies enhance AI system robustness and alignment with user intent.

## 摘要  
- **目标**：解决AI系统中的关键问题，包括奖励黑客攻击、JSON格式错误、特权信息（PI）泄露和系统提示不明确，旨在提升系统的可靠性、安全性及对工作流约束的遵守。  
- **方法**：论文采用奖励黑客攻击缓解策略（例如优先考虑硬约束）、JSON验证技术、通过提示重设计和过滤处理PI泄露，并针对τ-Bench和Travel Planner工作流进行系统提示工程。  
- **结果**：主要发现包括修改奖励逻辑以防止被利用的有效性、严格JSON格式规范的必要性、通过提示约束阻止PI泄露的可行性，以及明确系统提示对工作流对齐的重要性。  
- **关键贡献**：  
  1. 通过优先考虑硬约束而非软约束，提出一种新颖的奖励黑客攻击应对方法。  
  2. 为工具调用设计结构化的JSON响应框架。  
  3. 通过提示重设计和输出过滤缓解PI泄露。  
  4. 为τ-Bench和Travel Planner定制系统提示，以强制执行特定工作流。  
- **结论**：论文指出，解决奖励黑客攻击、格式错误和PI泄露问题需要结合奖励结构调整、严格输出验证和提示工程。这些策略增强了AI系统的鲁棒性并使其更符合用户意图。
